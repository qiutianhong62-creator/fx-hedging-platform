# 企业外汇套保与风险分析平台——产品设计说明

> 文档版本：V0.3  
> 当前状态：产品想法沉淀稿  
> 最近更新：2026-07-16

## 1. 项目定位

本项目面向存在外币应收或应付敞口的企业，帮助用户理解不同外汇套保策略对最终人民币现金流和风险边界的影响。

平台的目标不是单独制作一个远期、期权或掉期计算器，而是建立一个统一的策略分析平台：客户可以根据同一笔外汇敞口建立多种策略，并在同一套市场情景下比较其收益、成本和风险。

当前版本优先从“出口企业未来收到美元、最终兑换为人民币”的远期结汇案例开始。完整平台还必须支持“进口企业未来需要支付美元、提前锁定人民币购汇成本”的远期购汇案例。初始报价口径为人民币/美元（CNY per USD）。

本项目为教学和项目展示用途，不提供银行正式报价，也不构成投资、交易或套保建议。

## 2. 已确定的核心产品思想

### 2.1 产品不等于策略

- **产品**：远期、期权、期货、掉期、定存或货币市场套保等基础金融工具。
- **策略**：客户为了管理一笔敞口而采用的一套方案，可以包含一个或多个产品。
- **策略组成项**：一项具体交易，例如“用远期覆盖敞口的40%”或“用期权覆盖敞口的30%”。

因此，平台不能把“套保50%、80%、100%”直接当作最终的策略分类。它们只是远期策略内部的不同配置比例。

### 2.2 客户自主建立和比较策略

客户未来应当能够：

1. 建立一笔或多笔企业外汇敞口。
2. 点击“新增策略”。
3. 选择远期、期权、期货、掉期等产品。
4. 设置比例、期限、远期价、执行价、期权费等参数。
5. 给策略命名，例如“保守方案”“平衡方案”或“银行方案A”。
6. 勾选希望比较的策略。
7. 在同一张图和同一张表中查看比较结果。

### 2.3 支持复合策略

策略既可以是单一产品策略，也可以是复合策略。例如：

- 不套保100%。
- 远期套保50%＋未套保50%。
- 分层远期：40%按一个期限和汇率成交，30%按另一个期限和汇率成交。
- 远期50%＋期权30%＋未套保20%。
- 买入保护性期权＋卖出另一执行价期权，形成区间策略。
- 短期掉期＋长期远期。
- 货币市场套保＋期权。

系统应当先计算每个组成项的现金流和成本，再汇总为整套策略的最终结果。

### 2.4 远期产品包含两个业务方向

- **远期结汇**：适用于未来将收到外币的企业。企业约定未来卖出外币、买入人民币，主要目标是锁定人民币收入。
- **远期购汇**：适用于未来需要支付外币的企业。企业约定未来买入外币、卖出人民币，主要目标是锁定人民币成本。

两个方向使用相同的远期产品框架，但敞口方向、现金流符号、风险解释和评价指标不同，不能只通过修改页面文字来处理。

## 3. 核心用户流程

```text
建立企业敞口
    ↓
建立策略A、策略B、策略C……
    ↓
为每个策略添加一个或多个产品组成项
    ↓
统一市场情景和计算日期
    ↓
分别计算每个组成项的现金流、费用和收益
    ↓
汇总为策略级结果
    ↓
在同一张图和对比表中展示
```

## 4. 跨产品策略对比页面

现有固定比较“不套保、50%、80%、100%和自定义比例”的页面不是最终形态。未来应改为客户选择策略后进行跨产品比较。

### 4.1 页面主要区域

1. **企业敞口区域**
   - 收款或付款方向
   - 币种
   - 金额
   - 预计日期
   - 企业本位币

2. **策略建立区域**
   - 新增、复制、重命名或删除策略
   - 为策略添加产品组成项
   - 设置各组成项的金额、比例、期限和产品参数
   - 显示总覆盖比例和未覆盖比例

3. **策略选择区域**
   - 勾选需要放入图中的策略
   - 支持单一产品策略与复合策略同时比较

4. **统一收益图**
   - 每种策略对应一条曲线
   - 颜色和线型保持一致
   - 显示当前观察汇率和关键拐点

5. **策略对比表**
   - 汇总收入、成本、最差情景和风险指标
   - 解释每种策略的保护程度与机会成本

### 4.2 示例

同一张图中可以同时显示：

- 不套保
- 全额远期
- 保护性期权
- 外汇掉期
- 50%远期＋50%期权
- 客户自定义组合策略

原有的50%、80%和100%远期比较可以保留，但应作为“远期策略内部分析”的子功能，而不是平台的主策略对比逻辑。

## 5. 统一收益图的计算口径

为了公平比较不同产品，所有策略必须使用相同的基础条件。

### 5.1 坐标定义

- **横轴**：共同到期日的即期汇率，初始案例使用人民币/美元。
- **纵轴（外币应收/结汇）**：企业最终获得的人民币净收入，通常越高越有利。
- **纵轴（外币应付/购汇）**：企业最终支付的人民币净成本，通常越低越有利。

纵轴需要计入与策略相关的期权费、融资成本、交易成本或其他现金支出。同一张图只比较相同敞口方向的策略；如果未来需要同时汇总应收与应付敞口，应统一转换为相对基准的净损益或企业净现金流。

### 5.2 统一比较条件

- 相同的企业敞口金额和方向。
- 相同的基础币种和报价口径。
- 相同的到期日或统一估值日。
- 不同日期的现金流需要折现或滚动至同一比较日期。
- 费用和收益不能重复计算。
- 掉期的近端和远端现金流必须同时计入。

### 5.3 不同策略的典型曲线

- **不套保**：随到期汇率线性变化。
- **部分远期**：仍为直线，但斜率低于不套保。
- **全额远期**：在简化案例中接近水平线。
- **保护性期权**：出现收益拐点，限制不利方向损失，同时保留部分有利方向收益。
- **区间或领式策略**：可能同时出现下方保护和上方收益边界。
- **复合策略**：由多个产品收益函数相加形成新的曲线。
- **掉期策略**：需要在统一日期汇总近端、远端和资金占用后再进入比较。

## 6. 策略对比指标

每个策略至少展示以下指标：

| 指标 | 含义 |
|---|---|
| 总覆盖比例 | 已使用产品覆盖的敞口占比 |
| 未覆盖比例 | 仍随市场汇率变化的敞口占比 |
| 当前情景人民币收入/成本 | 指定到期汇率下的最终净现金流 |
| 最差情景结果 | 应收场景的最低收入，或应付场景的最高成本 |
| 最好情景结果 | 应收场景的最高收入，或应付场景的最低成本 |
| 现金流波动区间 | 情景区间内最好与最差结果之差 |
| 前期成本 | 期权费、手续费或其他初始支出 |
| 融资或资金成本 | 货币市场套保、掉期等产生的成本 |
| 相对不套保差额 | 与不采用套保时的收入差额 |
| 盈亏平衡点 | 策略相对基准发生收益关系变化的汇率 |

“收入最稳定”不等于“任何时候都最好”。平台应解释风险与机会之间的取舍，而不是只给出一个简单的最优推荐。

## 7. 产品模块规划

### 7.1 远期

远期模块必须同时支持远期结汇与远期购汇。

#### 7.1.1 远期结汇

适用场景：出口企业或其他未来将收到外币的客户。

- 企业未来卖出外币、买入人民币。
- 套保目标是锁定人民币收入，防止外币贬值导致结汇收入下降。
- 已覆盖部分的人民币收入为：外币名义金额 × 远期结汇汇率。
- 未覆盖部分按到期即期汇率结汇。

#### 7.1.2 远期购汇

适用场景：进口企业、外币借款偿还方或其他未来需要支付外币的客户。

- 企业未来买入外币、卖出人民币。
- 套保目标是锁定人民币成本，防止外币升值导致购汇成本上升。
- 已覆盖部分的人民币成本为：外币名义金额 × 远期购汇汇率。
- 未覆盖部分按到期即期汇率购汇。

#### 7.1.3 共同输入

- 名义金额或覆盖比例
- 远期汇率
- 到期日
- 业务方向：远期结汇或远期购汇
- 企业敞口方向：外币应收或外币应付

当前功能已经支持远期结汇的基础收入和情景曲线。后续需要加入远期购汇，并支持多个远期组成项、不同汇率和不同期限。

### 7.2 期权

第一阶段先实现到期收益，不急于加入复杂定价模型。

期权模块必须同时区分两个维度：看涨或看跌，以及买入或卖出。因此共有买入看涨、卖出看涨、买入看跌和卖出看跌四类基础头寸。

对于美元应收敞口，基础保护方式是买入美元看跌期权；对于美元应付敞口，基础保护方式是买入美元看涨期权。卖出期权收取期权费，同时产生履约义务，系统必须单独显示风险提示。

主要输入：

- 期权方向和类型
- 名义金额或覆盖比例
- 执行价
- 期权费
- 到期日

后续可扩展：

- Black-Scholes定价
- Delta、Gamma、Vega、Theta和Rho
- 蒙特卡洛模拟
- 美式期权二叉树模型
- 波动率和利率情景分析

定价分析可以作为期权产品的独立详情页；跨策略比较页面主要使用统一的到期净现金流。

完整规则、公式和组合策略见：[外汇期权模块设计](期权模块设计.md)。

### 7.3 外汇掉期

主要输入：

- 近端买卖方向和汇率
- 远端买卖方向和汇率
- 名义金额
- 近端日期和远端日期
- 掉期点或资金成本

掉期不能只画一条简单的到期收益线，必须完整记录近端和远端现金流，并统一到比较日期。

### 7.4 期货

主要输入：

- 合约数量与合约规模
- 期货价格
- 到期日
- 保证金和可能的资金占用
- 基差或结算方式

### 7.5 定存和货币市场套保

主要输入：

- 借入或存入金额
- 两种货币的利率
- 存贷款期限
- 即期兑换汇率
- 手续费和资金机会成本

## 8. 组合策略数据结构

建议采用三层结构：

```text
敞口 Exposure
  └─ 策略 Strategy
       ├─ 组成项 Leg 1：远期
       ├─ 组成项 Leg 2：期权
       ├─ 组成项 Leg 3：掉期
       └─ 自动计算的未覆盖部分
```

每个策略组成项至少需要保存：

- 产品类型
- 名义金额或配置比例
- 币种和交易方向
- 起始日和到期日
- 产品特有参数
- 前期成本
- 后续现金流

策略引擎不应把产品写死在一个大公式里。每个产品模块负责计算自己的现金流，组合引擎只负责校验、汇总和比较。

## 9. 必须加入的风险检查

- 总覆盖比例是否超过100%。
- 是否存在同一敞口被重复覆盖。
- 产品方向是否与企业敞口方向相反。
- 应收敞口是否错误使用远期购汇，应付敞口是否错误使用远期结汇。
- 币种是否匹配。
- 产品期限是否与敞口期限匹配。
- 是否遗漏期权费、手续费或融资成本。
- 是否存在不同日期现金流直接相加的问题。
- 使用的汇率报价方向是否一致。
- 市场数据是否过期或缺失。

如果允许企业主动过度套保，系统也应明确标记其已经超出风险对冲范围，可能形成投机头寸。

## 10. 当前版本与后续调整

### 10.1 当前已完成

- 单笔美元应收敞口案例。
- 远期结汇基础计算。
- 任意远期套保比例。
- 远期与不套保收益曲线。
- 远期内部的多比例情景比较原型。
- 初步组合策略数据结构。
- 过度套保检查和自动化测试。

### 10.2 后续需要重构

- 增加外币应付敞口和远期购汇计算。
- 根据应收或应付方向切换“人民币收入”和“人民币成本”的评价逻辑。
- 将当前固定的比例对比页面改为跨产品策略对比页面。
- 将50%、80%和100%比较移动到远期产品内部。
- 增加客户自定义策略及策略组成项管理。
- 允许用户选择哪些策略出现在同一张图中。
- 统一所有产品的现金流、费用和比较日期。

## 11. 推荐开发顺序

### 阶段一：冻结规则和案例

- 整理产品字段和公式。
- 分别建立远期结汇与远期购汇的标准案例。
- 建立标准测试案例及人工计算答案。
- 确定跨产品比较的统一现金流口径。

### 阶段二：策略建立器

- 新增、复制和删除策略。
- 一个策略包含多个组成项。
- 先接入远期和未套保组成项。
- 支持客户选择需要比较的策略。

### 阶段三：基础期权模块

- 先实现到期收益、执行价和期权费。
- 将期权策略加入统一收益图。
- 再逐步增加定价模型和Greeks。

### 阶段四：掉期、期货和货币市场套保

- 建立多日期现金流模型。
- 统一折现、融资成本和比较日期。
- 加入复合策略。

### 阶段五：企业级扩展

- 多笔敞口和多币种。
- 不同到期日和滚动套保。
- 市场数据更新。
- 报告导出和策略留痕。

## 12. 项目申请价值

本项目能够同时体现：

- 企业外汇风险管理问题的理解。
- 金融产品现金流和收益函数的建模。
- 模块化软件架构。
- 情景模拟和数据可视化。
- 用户输入校验和风险提示。
- 自动化测试和可解释性设计。

项目的核心亮点不是预测未来汇率，而是把复杂的企业套保产品转化为可以建立、组合、比较和解释的软件系统。

## 13. 当前已确认的产品决策

- 首个案例以出口企业美元应收和远期结汇为主，但产品架构必须同时支持进口企业美元应付和远期购汇。
- 远期比例分析保留，但不是主策略比较方式。
- 主页面未来进行跨产品、跨策略比较。
- 单一产品策略和复合策略都使用同一套比较引擎。
- 不同策略应放在同一张收益图中。
- 所有策略必须使用统一敞口、日期和净现金流口径。
- 先实现收益和现金流，再逐步增加复杂定价模型。
- 平台提供解释和风险边界，不直接替客户作出交易决定。
