# 第三阶段：不套保概率分析设计

> 日期：2026-07-18  
> 状态：待用户审阅书面规格  
> 项目：`fx-hedging-platform`

## 1. 阶段目标

第三阶段在第二阶段单一假设汇率计算之上，增加一个可独立测试的概率计算引擎。它回答：

> 在一个假设的未来汇率分布下，如果用户完全不套保，预计会获得或支付多少人民币，金额通常会落在哪些范围，以及达到目标金额的概率是多少？

这一阶段使用手工输入的预计到期汇率和年化波动率验证计算链条。它们是开发测试参数，不是普通用户最终需要填写的内容，也不是市场预测。后续阶段将分别用机构预测和历史汇率自动生成这两个参数。

## 2. 范围

### 2.1 本阶段支持

- 货币对固定为 `USD/CNY`。
- 敞口类型继续支持美元应收、美元应付和持有美元。
- 使用对数正态分布描述未来到期汇率。
- 接收一个假设的预计到期汇率和一个假设的年化波动率。
- 根据到期日自动把年化波动率转换为到期期限内的波动率。
- 计算不套保的预计人民币金额。
- 返回 50% 常见范围和 90% 较宽范围。
- 用户填写目标金额时，直接计算达标与未达标概率。
- 用户未填写目标金额时，不生成目标概率。
- 复用第二阶段的不套保单情景金额计算逻辑，不复制金额公式。
- 保留独立的分布模型边界，以便未来增加厚尾模型或经验分布。

### 2.2 本阶段不支持

- 抓取外部机构预测、即期汇率或历史汇率。
- 自动估算预计到期汇率和波动率。
- 将测试参数展示给最终普通用户填写。
- 远期、期权、定存或自由组合策略。
- 多币种、多到期日或路径相关分析。
- 把概率结果表述为保证、承诺、绝对最好或绝对最坏结果。

## 3. 模型选择

本阶段采用对数正态分布，不采用直接作用在汇率数值上的普通正态分布。原因是：

- 汇率始终保持为正数，不会出现负汇率。
- 模型基于相对变化，适合表达金融价格波动。
- 可以直接计算分位数与目标概率，不需要逐点枚举汇率。
- 第一版可以只使用 Python 标准库完成，减少额外依赖。

更复杂的厚尾分布能更好地表达极端行情，但需要更多参数、历史校准和回测，因此不进入本阶段。分布计算将放在独立服务中，未来可以替换模型而不改变金额计算和 API 的主要业务含义。

## 4. 用户可理解的数据流

```text
用户已有输入
（金额、到期日、应收/应付、可选目标）
                    +
开发测试参数
（预计到期汇率、年化波动率）
                    │
                    ▼
          生成未来汇率概率分布
                    │
                    ▼
        计算预计金额和概率金额范围
                    │
                    ▼
        计算达标概率与未达标概率
```

正式版本中的参数来源将替换为：

```text
公开机构预测 ──→ 预计到期汇率
历史汇率数据 ──→ 年化波动率
                      │
                      ▼
              同一个概率计算引擎
```

## 5. 输入含义与期限换算

新增两个开发测试参数：

- `assumed_expected_maturity_spot`：假设的预计到期汇率，必须是有限且大于 0 的数。
- `assumed_annualized_volatility_pct`：假设的年化波动率百分数，必须是有限且大于 0 的数；例如 `5` 表示年化波动率 5%，不是 500%。

到期期限按自然日计算：

```text
期限年数 = 距离到期的自然日数 ÷ 365
到期波动率 = 年化波动率 × √期限年数
```

预计到期汇率被定义为分布的数学期望。内部对数正态参数需要做均值修正，保证概率分布的平均汇率与该输入一致，而不是把它误当成对数分布的中位数。

服务内部允许显式传入估值日，生产 API 默认使用服务器当天日期，自动测试使用固定估值日。这样期限计算既符合实际日期，也能稳定复现测试结果。

## 6. 输出结果

后端返回四类业务结果。

### 6.1 核心预测

- 预计到期汇率：等于输入的假设预计到期汇率。
- 预计不套保人民币金额：美元金额乘以预计到期汇率。
- 结果含义继续区分 `cny_proceeds` 和 `cny_cost`。

因为不套保金额与汇率是线性关系，所以预计金额等于美元金额乘以预计汇率，不需要使用模拟取平均。

### 6.2 50% 常见范围

使用汇率分布的第 25 百分位和第 75 百分位作为边界：

```text
P25 到 P75 = 中间 50% 的可能结果
```

分别调用第二阶段金额计算器，把两个边界汇率转换成人民币金额。

### 6.3 90% 较宽范围

使用汇率分布的第 5 百分位和第 95 百分位作为边界：

```text
P05 到 P95 = 中间 90% 的可能结果
```

这两个范围是概率参考区间，不叫“最好”和“最坏”。区间之外仍然存在小概率结果。

### 6.4 业务方向

API 的区间边界始终按数值从低到高返回，避免同一字段因业务类型而改变含义。`result_kind` 告诉后续前端怎样解释好坏：

```text
美元应收或持有美元：金额越高越有利
美元应付：金额越低越有利
```

## 7. 目标达成概率

用户填写目标金额后，系统先反算临界到期汇率：

```text
临界汇率 = 目标人民币金额 ÷ 美元金额
```

然后直接使用对数正态分布的累计概率函数计算曲线面积，不逐个枚举 6.50、6.51、6.52 等汇率点。

### 7.1 美元应收与持有美元

目标表示至少希望获得的人民币金额：

```text
达标概率   = P（到期汇率 ≥ 临界汇率）
未达标概率 = P（到期汇率 < 临界汇率）
```

### 7.2 美元应付

目标表示可以接受的最高人民币成本：

```text
达标概率   = P（到期汇率 ≤ 临界汇率）
未达标概率 = P（到期汇率 > 临界汇率）
```

连续分布中单独一个精确汇率点的概率为 0，因此边界使用大于还是大于等于不会改变数值结果。

达标概率和未达标概率在未舍入前必须互为补数，总和为 1。用户未填写目标金额时，目标概率结果返回 `null`。

## 8. 后端结构

第三阶段继续使用模块化单体 FastAPI 后端：

```text
POST /api/v1/analysis/no-hedge/probability
                         │
                         ▼
                请求模型与输入校验
                         │
                         ▼
              对数正态汇率分布服务
              ├─ 期限换算
              ├─ 分位数
              └─ 累计概率
                         │
                         ▼
              不套保概率分析服务
              ├─ 复用单情景金额计算
              ├─ 生成金额区间
              └─ 计算目标概率
                         │
                         ▼
                  结构化响应模型
```

组件边界：

- 请求与响应模型：定义 API 接收和返回的稳定字段。
- 分布服务：只处理汇率概率，不理解应收、应付或人民币目标。
- 概率分析服务：理解业务方向，组合分位数、金额和目标概率。
- 第二阶段单情景服务：继续负责人民币金额、舍入和结果类型。
- API 路由：只负责连接请求和服务，不放置概率公式。

## 9. 接口契约

### 9.1 请求

`POST /api/v1/analysis/no-hedge/probability`

```json
{
  "currency_pair": "USD/CNY",
  "exposure_type": "usd_receivable",
  "notional_usd": 1000000,
  "maturity_date": "2030-12-31",
  "target_cny": 6800000,
  "assumed_expected_maturity_spot": 6.80,
  "assumed_annualized_volatility_pct": 5.0
}
```

请求沿用第一、第二阶段的输入规则，并增加两个必填测试参数。字段名必须保留 `assumed`，避免把测试输入误解为真实预测。

### 9.2 响应

响应的字段形状如下；示例数值只说明结构，不作为固定验收答案：

```json
{
  "status": "calculated",

    "model_type": "lognormal",

    "assumed_expected_maturity_spot": 6.80,
    "assumed_annualized_volatility_pct": 5.0,
    "horizon_days": 365
  },
  "result_kind": "cny_proceeds",
  "expected_result": {
    "spot": 6.80,
    "amount_cny": 6800000.00
  },
  "typical_range_50": {
    "probability": 0.50,
    "lower_spot": 6.57,

    "lower_amount_cny": 6570000.00,

```

概率字段使用 `0` 到 `1` 的数值表达，前端未来负责显示为百分比。金额继续按人民币分四舍五入。区间汇率保留足够计算精度，显示位数由未来前端决定。

## 10. 数字精度与计算方式

- 汇率分布、平方根、对数、指数、分位数和累计概率使用 Python 浮点数学与标准库正态分布工具。
- 正式人民币金额继续交给第二阶段的 `Decimal` 计算路径并按分舍入。
- 概率通过解析累计分布函数计算，不使用随机抽样，因此相同输入必须得到相同结果。
- 本阶段不引入 SciPy、NumPy 或蒙特卡洛模拟。
- 预计金额、分位数金额和临界汇率应在输出前检查为有限值；无效数学结果不能静默返回。

## 11. 错误处理

除现有输入错误外，本阶段新增稳定中文错误：

```text
assumed_expected_maturity_spot ≤ 0 或不是有限数
→ 假设预计到期汇率必须是大于 0 的有效数字

assumed_annualized_volatility_pct ≤ 0 或不是有限数
→ 假设年化波动率必须是大于 0 的有效数字
```

第一阶段已有规则继续生效：

- 货币对必须是 `USD/CNY`。
- 敞口类型必须受支持。
- 美元金额必须大于 0。
- 到期日必须晚于今天。
- 目标金额若填写则必须大于 0。

数学计算若因超出机器可表示范围而失败，API 返回稳定的计算错误，不返回 `NaN`、无穷大或部分结果。第一版不设置主观的“正常波动率”上限，以免把压力测试误判为输入错误。

## 12. 测试与验收

### 12.1 分布服务测试

- 汇率分位数严格满足 `P05 < P25 < P50 < P75 < P95`。
- 50% 区间由 P25 与 P75 组成，90% 区间由 P05 与 P95 组成。
- 输入的预计汇率确实是分布数学期望。
- 年化波动率按到期天数的平方根正确缩放。
- 同一预计汇率下，到期越远或年化波动率越高，概率范围越宽。
- 累计概率始终位于 0 与 1 之间。
- 分布服务使用固定估值日时结果可重复。

### 12.2 业务服务测试

- 美元应收与持有美元使用右侧概率作为达标概率。
- 美元应付使用左侧概率作为达标概率。
- 应收或持有美元的目标越高，达标概率越低。
- 应付的最高成本目标越高，达标概率越高。
- 达标概率与未达标概率互为补数。
- 未填写目标时返回 `target_probability: null`。
- 两个金额范围按数值从低到高排列。
- 金额计算和人民币分舍入继续复用第二阶段逻辑。
- 原有第一、第二阶段测试继续通过。

### 12.3 API 和手工验收

通过 FastAPI `/docs` 页面输入一笔敞口、测试预计汇率和测试波动率，能够看到：

- 明确标记为假设、不是市场预测的分布元数据。
- 预计不套保人民币金额。
- 50% 常见金额范围。
- 90% 较宽金额范围。
- 可选的达标与未达标概率。
- 不合理输入对应的中文错误。

## 13. 后续连接方式

后续阶段不改动本阶段的主要输出含义，只替换参数来源：

```text
第四阶段：历史 USD/CNY 数据 ──→ 自动年化波动率
第五阶段：公开机构预测      ──→ 多个到期汇率预测
第六阶段：预测汇总与期限转换 ──→ 概率分布参数
第七阶段：接入套保策略组合   ──→ 策略金额与不套保基准比较
```

因此，第三阶段是独立的概率计算基础，但自身不声称预测市场，也不涉及外部数据抓取。
