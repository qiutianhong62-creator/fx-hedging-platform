# 企业外汇策略概率与风险分析后端设计

> 日期：2026-07-18  
> 状态：已完成讨论，等待用户审阅书面规范  
> 项目：`fx-hedging-platform`

## 1. 产品目标

第一版面向存在真实美元敞口的用户。用户输入美元敞口、统一到期日、目标人民币金额，以及银行提供的真实远期或期权报价。系统结合公开金融机构预测和 USD/CNY 历史数据，建立到期汇率概率分布，计算不套保及用户自由组合策略的到期结果、达标概率和风险。

系统用于风险分析和教学展示，不提供银行正式报价，不承诺预测准确性，也不直接告诉用户必须购买哪种产品。

## 2. 第一版范围

### 2.1 支持范围

- 货币对：USD/CNY，统一解释为 1 美元对应多少人民币。
- 真实敞口：未来美元应收、未来美元应付、当前持有美元。
- 所有敞口和交易使用同一个到期日。
- 产品：远期结汇、远期购汇，以及买入或卖出的美元看涨、看跌欧式期权。
- 用户输入银行真实报价：远期汇率、期权执行价、期权费和名义金额。
- 系统在用户真实报价旁提供可追溯的市场参考远期价和参考期权价，但明确标注其不是可成交的银行报价。
- 用户可自由组合多笔远期和期权。
- 分析方式：只分析统一到期日的最终现金流，不计算到期前的每日市值。
- 比较对象：不套保、用户策略、100% 远期和标准保护性期权。

### 2.2 第一版不支持

- USD/CNY 之外的货币对。
- 同一分析中混合不同到期日。
- 没有真实外币敞口的纯投机用户。
- 每日盯市、提前平仓、美式期权提前行权和实时交易。
- 卖出期权的具体保证金、授信占用和追加保证金计算。
- AI 直接决定用户应购买的产品或金额。

卖出期权仍可参与到期损益分析，但结果必须提示实际交易可能需要保证金或授信，当前分析未计入具体资金占用。

## 3. 用户分析流程

```text
输入真实美元敞口、金额和到期日
              │
              ▼
获取公开机构预测与历史 USD/CNY 数据
              │
              ▼
生成用户到期日对应的汇率概率分布
              │
              ▼
展示不套保的预计金额、最可能区间和概率边界
              │
              ▼
用户填写目标金额及真实远期/期权报价
              │
              ▼
计算用户自由组合策略的到期结果
              │
              ▼
展示达标概率、风险、压力测试和基准比较
```

目标金额由用户填写。系统未来可以提供可修改的建议值，但建议值算法不属于第一轮实施范围。

## 4. 后端总体结构

第一版采用模块化单体 Python 后端。前端只负责输入和展示，正式计算全部由后端完成。

```text
React 前端
    │
    ▼
FastAPI 统一入口
    ├─ 输入与任务模块
    ├─ 市场数据模块
    ├─ 预测概率模块
    ├─ 策略计算模块
    ├─ 风险分析模块
    └─ 档案与数据库模块
```

各模块位于同一后端应用中，但通过明确的数据接口连接，可以独立理解和测试。现有 `python/hedging/` 作为正式计算引擎的起点；现有网页端 TypeScript 计算在迁移期间仅用于对照，最终以后端结果为准。

## 5. 核心数据对象

### 5.1 敞口 Exposure

- 敞口类型：美元应收、美元应付或持有美元。
- 美元金额。
- 统一到期日。
- 人民币目标金额。

### 5.2 策略 Strategy

一个敞口可以建立多个待比较策略。一个策略包含任意数量的交易组成项。

### 5.3 交易组成项 Trade Leg

远期组成项保存：

- 远期结汇或远期购汇。
- 美元名义金额。
- 用户真实远期汇率。
- 到期日。

期权组成项保存：

- 看涨或看跌。
- 买入或卖出。
- 美元名义金额。
- 执行价。
- 期权费率或期权费总额及币种。
- 到期日。

### 5.4 市场预测快照 Forecast Snapshot

- 当前 USD/CNY。
- 使用的机构预测及原文链接。
- 机构预测发布日期和对应期限。
- 用户到期日匹配方式。
- 使用的历史数据区间。
- 机构与历史模型的实际权重。
- 最终汇率概率分布。
- 预测可信度及原因。

### 5.5 分析任务 Analysis Run

- 原始请求。
- 等待、取数、预测、计算、完成或失败状态。
- 开始和完成时间。
- 外部来源失败记录。
- 使用的模型和计算规则版本。
- 最终分析结果或错误原因。

## 6. 市场预测与概率模型

### 6.1 公开机构预测

第一版只使用无需登录、公开可访问且能保留原文链接的机构材料。信息发现工具优先通过 AnySearch MCP 接入，但后端使用统一搜索接口，不与某一种搜索工具绑定。

搜索和提取层输出统一字段：机构、标题、原文链接、发布日期、货币对、预测值、预测对应日期和支持该预测的原文摘录。

机构材料进入模型前必须验证：

- 确实针对 USD/CNY，而不是 USD/CNH 或其他报价。
- 内容属于明确预测而非一般讨论。
- 发布日期和预测期限明确。
- 原文可追溯且未过期。
- 数值未超出基本合理性检查。

AI 可以帮助从网页或 PDF 中提取信息，但不能直接计算正式金额或绕过规则验证。

### 6.2 到期日匹配

- 有直接对应日期的预测时使用原始值。
- 用户日期位于同一机构两个预测期限之间时，按时间插值，并标记为系统估算值。
- 只有一个邻近期限时可以降低权重后使用。
- 超出机构有效预测范围时不进行长距离外推，该机构在本次日期匹配中被排除。

### 6.3 历史模型与降级

机构预测主要决定汇率分布中心，历史 USD/CNY 数据主要决定分布宽度和尾部波动。有效机构越少，历史模型权重越高。

- 机构数据充足：机构预测为主，历史模型辅助。
- 机构数据较少：两者共同决定分布，并降低可信度。
- 没有有效机构预测：仅使用历史模型并明确标注。
- 历史数据也不足：停止输出概率，只提供固定汇率压力测试。

具体权重数值需要通过历史检验确定，不在设计中预先写死。

### 6.4 概率输出

- 综合预测中心。
- 最可能金额和最可能区间。
- 预计金额。
- 较好情景和较差情景的概率边界。
- 预测可信度及形成原因。

不使用绝对的“最好”和“最坏”措辞。第一版采用明确标注的概率分位结果，并另外提供不参与正常概率计算的压力测试。

### 6.5 市场参考报价

用户策略始终使用用户输入的真实银行报价计算。系统另外生成市场参考值，帮助用户判断真实报价大致处于什么位置：

- 参考远期价优先根据可追溯的公开远期市场数据生成；缺少直接数据时，可以根据当前即期汇率和同期限中外利率计算理论参考值。
- 参考期权价优先使用可追溯的公开期权或波动率数据；缺少隐含波动率时，可以使用历史波动率生成模型参考值，并降低可信度。
- 参考值必须保存计算日期、数据来源、期限、模型和输入参数。
- 参考值不替换用户真实报价，也不表述为银行可成交价格。

100% 远期和标准保护性期权基准优先使用上述市场参考值。参考数据不足时，系统省略无法公平定价的标准方案，而不是编造报价。

## 7. 策略计算引擎

计算引擎接收一个到期汇率，分别计算原始敞口和每笔交易的到期现金流，再汇总整个策略。

```text
最终结果
= 原始美元敞口的人民币现金流
+ 所有远期到期现金流
+ 所有期权到期现金流
- 支付的期权费
+ 收到的期权费
```

系统自动生成约 200 至 500 个足够密集的汇率情景，并强制加入期权执行价、目标金额临界点和其他收益拐点。汇率范围根据本次概率模型自动确定，再额外加入压力情景。市场数据只获取一次，不会为每个汇率点重复抓取。

计算量与“情景数量 × 交易组成项数量”成正比，对第一版规模属于轻量计算。前端不展示全部内部情景，只展示曲线和汇总结果。

### 7.1 三本账

- 敞口账：用户原本存在多少美元风险。
- 义务账：远期和卖出期权在各情景下要求交付或购买多少美元。
- 现金账：原始业务现金流、产品损益、期权费及最终人民币结果。

### 7.2 自由组合诊断

名义金额合计超过敞口时不直接禁止。系统计算净现金流和净义务，并诊断：

- 完全或部分抵消。
- 重复覆盖或潜在过度套保。
- 额外方向性风险。
- 潜在履约金额超过真实敞口。
- 下跌保护被抵消。
- 上涨收益被限制。
- 可识别的合成结构，例如相同条款下远期结汇加卖出看跌大致形成卖出看涨敞口。

## 8. 风险分析与比较

### 8.1 不套保分析

- 最可能金额和区间。
- 预计金额。
- 较好和较差概率情景。
- 达到目标与未达到目标的概率。
- 未达标时的平均缺口。

### 8.2 用户策略分析

- 预计最终金额和最可能区间。
- 较好和较差概率情景。
- 达标与未达标概率。
- 未达标平均缺口和较严重情景缺口。
- 前期期权费和其他已输入成本。
- 风险来源按原始敞口及各交易组成项拆分。

### 8.3 公平比较

用户策略与不套保、100% 远期和标准保护性期权使用同一敞口、到期日、目标金额和概率分布。比较内容包括达标概率、预计金额、较差情景、较好情景、前期成本、结果波动和机会成本。

系统解释取舍，不宣称存在唯一最优策略。

### 8.4 压力测试

独立测试 USD/CNY 上涨和下跌 5%、10%，以及基于历史数据定义的极端变化。压力结果不并入正常概率分布。

## 9. 错误处理和运行进度

### 9.1 阻止计算的错误

- 非正金额或无效目标金额。
- 币种或报价方向不一致。
- 敞口、远期和期权到期日不一致。
- 无效的远期汇率、执行价或期权费。
- 无法确定的交易方向。

### 9.2 可降级的数据问题

- 单一机构失败：排除该来源并记录原因。
- 机构数量不足：提高历史模型权重并降低可信度。
- 所有机构失败：仅使用历史模型并显著标注。
- 概率模型不可用：仍允许执行固定汇率策略计算和压力测试。

### 9.3 运行进度

后端对长任务提供阶段状态：检查输入、获取预测、建立概率、计算策略、生成报告、完成或失败。第一版可以先使用查询任务状态的简单方式，确有实时展示需要时再增加流式推送。

## 10. 测试策略

### 10.1 公式测试

- 应收、应付和持有美元的基准现金流。
- 远期结汇和购汇。
- 买入或卖出的美元看涨、看跌期权。
- 期权费方向和组合求和。

### 10.2 组合测试

- 相反交易完全抵消。
- 部分抵消和合成结构识别。
- 潜在交付义务超过真实敞口。
- 目标金额临界点和达标概率。
- 不套保、用户策略和标准策略在相同市场假设下公平比较。

### 10.3 数据与流程测试

- 机构预测直接匹配、插值、邻近匹配和排除。
- 单一来源失败、全部机构失败和历史模型降级。
- 从提交请求、保存快照到返回完整报告的端到端流程。

每个阶段必须保留人工可复算的小案例，自动测试通过后再进入下一阶段。

## 11. 分阶段实施与验收

不得一次性生成全部后端。每一阶段都必须能够独立运行、测试并向用户解释，再进入下一阶段。

### 阶段一：后端入口与输入检查

- 建立 FastAPI 最小应用。
- 定义敞口、目标和统一到期日输入格式。
- 返回整理后的输入或易懂的错误提示。
- 不进行策略、市场或概率计算。

### 阶段二：到期策略计算

- 迁移并统一远期和四类期权到期现金流。
- 支持多笔交易自由组合、期权费和不套保基准。
- 使用人工提供的固定汇率情景验证公式。

### 阶段三：风险分析

- 使用固定的临时汇率概率数据。
- 计算预计金额、概率分位、达标概率、未达标平均缺口及基准比较。

### 阶段四：历史汇率模型

- 接入真实 USD/CNY 历史数据。
- 生成历史波动和概率范围。
- 在机构预测缺失时能够独立完成基础分析。
- 在没有公开隐含波动率时，为期权市场参考值提供带有低可信度标记的历史波动率输入。

### 阶段五：机构预测自动采集

- 通过统一搜索接口接入 AnySearch MCP 或可替代工具。
- 完成公开机构材料提取、验证、日期匹配和历史模型融合。
- 接入生成参考远期价和参考期权价所需的公开市场数据；任何缺失都必须降级或省略相关基准，不得伪造。

### 阶段六：任务档案与数据库

- 保存输入、预测快照、运行状态、失败原因和最终结果。
- 支持查看同一任务当时使用的数据和模型版本。

### 阶段七：前后端正式连接

- 现有 React 页面改为调用后端。
- 展示运行进度、概率指标、比较结果和风险提示。
- 正式结果以后端计算为唯一依据。

每阶段采用同一验收节奏：实现小范围功能、运行自动测试、用人工案例复算、向用户解释结果、获得确认后再继续。

## 12. 成功标准

第一版完成时，用户可以输入一笔真实 USD/CNY 敞口、目标金额和同日到期的自由远期/期权组合。系统能使用可追溯的公开机构预测与历史模型生成概率分布，输出不套保和用户策略的预计金额、概率区间、达标概率、未达标缺口、压力测试和结构风险提示。任何结果都能追溯到用户输入、市场来源、模型版本和确定性现金流公式。
