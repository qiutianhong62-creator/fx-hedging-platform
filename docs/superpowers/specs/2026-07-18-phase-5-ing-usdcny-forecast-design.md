# 第五阶段：ING USD/CNY 到期汇率预测设计

> 日期：2026-07-18
> 状态：待用户审阅书面规格
> 项目：`fx-hedging-platform`

## 1. 阶段目标

第五阶段建立第一条可追溯的外部机构预测链路。系统自动读取 ING 官方 USD/CNY 季度预测，将用户输入的任意到期日匹配成一个预计到期汇率，并返回原始预测点、日期匹配方法、更新时间和原文链接。

本阶段回答：

> 对于用户选定的未来一年内到期日，根据 ING 公开预测和最新 FRED 市场参考值，系统估算的 USD/CNY 到期汇率是多少？这个结果来自 ING 原始预测，还是系统插值？

这是单一机构试验链路，不是多机构综合预测。第六阶段再将本阶段的预计到期汇率、第四阶段的历史波动率和第三阶段的概率引擎连接起来。

## 2. 范围

### 2.1 本阶段支持

- 货币对固定为 `USD/CNY`。
- 第一家机构固定为 ING，使用无需登录的官方固定预测页。
- 用户提供一个未来到期日。
- 到期日最多距服务器估值日 365 个自然日。
- 自动提取 ING FX 表格中 China / USD/CNY 行的季度预测点。
- 将季度标签转换成季度最后一个自然日。
- 到期日正好命中 ING 预测日时直接使用原始预测。
- 到期日位于两个 ING 预测点之间时，按自然日线性插值。
- 到期日位于最新 FRED 参考点与第一个未来 ING 预测点之间时，在两者之间插值。
- 返回来源列表，虽然第一版列表只有 ING。
- 返回匹配方式、所用锚点、ING 更新日期、官方链接、缓存状态和限制说明。
- 提供独立只读 API，可在 FastAPI `/docs` 中手工验证。

### 2.2 本阶段不支持

- 多家机构加权或一致预期。
- Wind、Bloomberg、Refinitiv 等付费数据或市场远期曲线。
- 超过未来一年的长距离预测。
- 在最后一个可用 ING 预测点之后沿用最后数值或外推。
- 从 ING 普通新闻评论中使用大模型自由推断数字。
- 自动发现 ING 每月新文章或解析 PDF。
- 修改现有不套保概率接口。
- 将单一 ING 结果声称为市场共识或多机构综合预测。
- 数据库、后台定时任务或分布式缓存。

## 3. 数据源

### 3.1 ING 官方预测表

固定来源：

```text
https://think.ing.com/forecasts/
```

选择原因：

- ING 是大型国际金融机构。
- 页面无需登录，原文可公开追溯。
- URL 固定，不需要每月猜测新文章地址。
- FX 表格使用结构化的季度列和货币对行。
- 当前已验证页面包含 `China | USD/CNY` 及多个季度末预测。

例如，当页面列名为 `3Q26F` 且 USD/CNY 数值为 `6.74` 时，解释为：

```text
机构：ING
货币对：USD/CNY
预测日：2026-09-30
预测值：6.74 CNY per 1 USD
```

### 3.2 FRED 市场参考锚点

复用第四阶段 `GET /api/v1/market/usd-cny/history-summary` 背后的市场数据服务，不再建第二套 FRED 客户端。

FRED 锚点必须使用它实际的 `observation_date`，不得把最近一期值伪装成估值日当日实时报价。第四阶段已经限制该日期不得落后估值日超过 14 个自然日。

## 4. 用户可理解的数据流

```text
用户输入到期日
        │
        ▼
 读取ING官方季度预测表
        │
        ▼
 清理并转换预测日期
        │
        ▼
 按用户到期日选择锚点
        │
   ┌────┼─────────┐
   ▼    ▼             ▼
直接命中  两个ING点之间  早于第一个ING点
   │    │             │
   │    │             ▼
   │    │      按需读取FRED参考锚点
   │    │             │
   └────┴──────┬─────────┘
                ▼
  返回预计汇率、锚点、方法和来源
```

FRED 是条件依赖：只有用户到期日早于第一个未来 ING 预测点时才读取。直接命中 ING 或在两个 ING 点之间插值时，FRED 暂时失败不得影响结果。

第五阶段不返回达标概率或金额区间；它只提供后续概率模型所需的“分布中心”。

## 5. 组件边界

新增组件各自只承担一种职责：

```text
ING客户端
  只负责请求固定官方页面

ING表格解析器
  只负责识别FX表、USD/CNY行、更新日期和季度点

到期日匹配器
  只负责直接命中或按自然日插值

ING预测缓存
  只负责保存已验证的机构预测快照

到期预测服务
  负责组织ING、按需读取FRED、新鲜度、日期匹配和返回结果
```

API 路由只调用到期预测服务，不包含网页解析、缓存或插值公式。

代码使用统一的“机构预测来源”边界。第一版实现 ING，以后加入 MUFG、汇丰或其他机构时，日期匹配器和公开 API 不需要重写。

## 6. ING 解析与验证规则

### 6.1 表格定位

解析器必须同时验证：

- 所在区域是 `FX`，不是 GDP、CPI 或其他预测表。
- 区域包含最后更新标记。
- 行内国家是 `China`。
- 货币对完整匹配 `USD/CNY`，不接受 `CNH`、`CNY/USD` 或模糊文本。
- 预测列符合季度标签，例如 `3Q26F`。
- 预测值是有限且大于 0 的数字。
- 预测日期严格递增，不得重复。
- 至少包含两个估值日之后的有效预测点，否则不足以完成任意日期匹配。

如果页面排版变化导致这些条件无法同时确认，整份新响应视为上游格式异常，不部分使用猜测出的数据。

### 6.2 季度日期转换

```text
1QyyF -> 20yy-03-31
2QyyF -> 20yy-06-30
3QyyF -> 20yy-09-30
4QyyF -> 20yy-12-31
```

第一版只接受两位年份且按 `2000 + yy` 转换。只保留对当前估值日与未来一年匹配有用的点，但缓存可保存本次解析到的全部有效 USD/CNY 点，便于追溯。

### 6.3 ING 更新日期

ING FX 区域必须包含可解析的 `Last updated` 日和月。如果页面没有明示年份：

1. 使用抓取日所在年和前一年生成两个候选日期。
2. 剔除晚于抓取日的候选。
3. 选择剩余候选中最新的日期。

该规则保证 1 月抓取、页面显示上年 12 月更新时不会误标为未来日期。更新日期晚于抓取日、无法解析或距估值日超过 45 个自然日时，预测不可用。

## 7. 到期日匹配规则

### 7.1 可用期限

```text
估值日 < 用户到期日 <= 估值日 + 365天
```

同时必须存在日期不早于用户到期日的 ING 预测点。即使用户日期在 365 天内，如果 ING 没有覆盖到该日期的后一个锚点，也返回“机构预测期限不足”，不外推。

### 7.2 直接命中

当用户到期日与某个 ING 预测日完全相同：

```text
预计到期汇率 = ING原始预测值
matching_method = direct
```

该结果可称为 ING 对该日期的原始预测。

### 7.3 自然日线性插值

对前锚点 `(d₀, S₀)`、后锚点 `(d₁, S₁)` 和用户日期 `d`：

```text
w = (d - d₀)的自然日数 / (d₁ - d₀)的自然日数

预计到期汇率 = S₀ + w × (S₁ - S₀)
matching_method = interpolated
```

插值必须使用完整精度，后端不为显示提前截断。

前锚点选择：

- 用户日期早于第一个未来 ING 点：前锚点使用 FRED 最近有效观察值及其实际观察日。
- 用户日期位于两个 ING 点之间：前后锚点均使用 ING 原始预测。

响应必须返回两个锚点的来源、日期和汇率。插值结果只能标记为系统估算，不得标记为 ING 对当天的直接预测。

## 8. 缓存与降级

### 8.1 缓存位置和内容

使用服务器本地 JSON 缓存，放在已被 Git 忽略的 `python/.cache/market-data/` 目录下，不引入数据库。

缓存包含：

- 机构名和货币对。
- 固定官方链接。
- ING 预测更新日期。
- 本次成功抓取的 UTC 时间。
- 已清理、按日期排序的原始预测点。

不把某个用户到期日的插值结果存入机构快照，因为不同用户的到期日不同。

### 8.2 新鲜度

- 抓取时间不超过 24 小时：正常复用缓存。
- 需要刷新且 ING 网络请求成功：验证后原子替换缓存。
- 只有网络或 HTTP 失败时，可以使用抓取时间不超过 7 天的旧缓存，并标记 `stale_fallback`。
- 无论新抓取还是缓存，ING 自身的预测更新日距估值日均不得超过 45 天。
- 未来时间缓存、损坏缓存、错误机构、错误货币对、非正汇率或日期无序均视为无缓存。

如果新页面能下载但格式验证失败，返回“ ING 预测格式异常”，不用超过 24 小时的缓存掩盖结构变化。

## 9. 可扩展的预测结构

公开响应从第一版就使用复数 `sources` 列表，不把 ING 字段散落在顶层。

第一版必须明确返回：

- `source_count: 1`
- `aggregation_status: single_source_trial`
- `is_consensus_forecast: false`
- `sources[0].institution: ING`

未来增加机构时，每家机构仍返回同样的名称、更新日、链接和预测点。多机构权重、分歧范围、可信度和观点理由属于后续阶段，第一版不返回空占位数值。

## 10. API 契约

### 10.1 请求

```text
GET /api/v1/forecasts/usd-cny/maturity-estimate
    ?maturity_date=YYYY-MM-DD
```

请求只包含到期日。货币对、机构和匹配方法不由用户修改，保证试验链路口径一致。

### 10.2 成功响应示例

数值只说明结构，不作为固定预测：

```json
{
  "status": "available",
  "currency_pair": "USD/CNY",
  "quote_convention": "CNY per 1 USD",
  "valuation_date": "2026-07-18",
  "maturity_date": "2026-11-15",
  "expected_maturity_spot": 6.72,
  "matching": {
    "method": "interpolated",
    "is_system_estimate": true,
    "day_weight": 0.5,
    "anchors": [
      {
        "source": "ING",
        "date": "2026-09-30",
        "spot": 6.74
      },
      {
        "source": "ING",
        "date": "2026-12-31",
        "spot": 6.70
      }
    ]
  },
  "source_count": 1,
  "aggregation_status": "single_source_trial",
  "is_consensus_forecast": false,
  "sources": [
    {
      "institution": "ING",
      "source_updated_date": "2026-07-16",
      "source_url": "https://think.ing.com/forecasts/",
      "forecast_points": [
        {"date": "2026-09-30", "spot": 6.74},
        {"date": "2026-12-31", "spot": 6.70}
      ],
      "cache_status": "live_fetch",
      "fetched_at_utc": "2026-07-18T10:00:00Z"
    }
  ],
  "limitations": [
    "This is a single-institution trial, not a consensus forecast.",
    "Interpolated values are system estimates, not direct ING forecasts."
  ]
}
```

`forecast_points` 应返回本次日期匹配使用的有效 ING 原始预测集，不只返回两个锚点，便于用户检查预测路径。

### 10.3 稳定错误

- `forecast_maturity_invalid` / 422：到期日不在估值日之后的未来 365 天内。
- `forecast_horizon_insufficient` / 503：ING 没有覆盖该到期日的后锚点。
- `forecast_source_unavailable` / 503：ING 网络失败且无可用缓存。
- `forecast_source_invalid` / 502：ING 页面结构或数值验证失败。
- `forecast_source_stale` / 503：ING 预测更新日距估值日超过 45 天。
- 只有近期到期日需要 FRED 锚点时，第四阶段 FRED 错误才继续使用已定义的稳定错误代码。直接 ING 命中和两个 ING 点之间的匹配不调用 FRED。

## 11. 测试策略

### 11.1 自动测试

自动测试不访问真实 ING 网站，使用固定 HTML 样例和模拟 HTTP 客户端。

必须覆盖：

- 只在 FX 区域识别 China / USD/CNY。
- 正确解析 ING 更新日和跨年年份。
- `1QyyF` 至 `4QyyF` 的季度末日期转换。
- 非 USD/CNY、反向报价、缺列、非数字、零、负数、重复或无序点被拒绝。
- 正好命中 ING 点时不进行插值。
- 两个 ING 点之间按自然日插值。
- 第一个 ING 点之前使用 FRED 实际观察日和参考值。
- 直接命中 ING 或在两个 ING 点之间时不请求 FRED。
- 到期日不合法、超过 365 天和后锚点不足。
- 24 小时正常缓存、7 天网络降级、未来缓存和损坏缓存。
- ING 更新日超过 45 天。
- 页面格式异常时不用旧缓存掩盖问题。
- API 成功响应包含来源、锚点、方法、限制和单一机构标记。
- 所有稳定错误代码和 HTTP 状态。
- 现有全部测试继续通过。

### 11.2 真实联网验收

完成后手工调用一次新 API：

1. 确认请求直接访问 ING 官方固定页。
2. 确认响应中的 USD/CNY 数字与当前官方表格一致。
3. 确认 ING 更新日可追溯且不超过 45 天。
4. 确认用户到期日的直接或插值结果可以用响应中锚点手工复算。
5. 第二次调用确认使用 24 小时缓存。

## 12. 安全、合规与限制

- 只读公开、无需登录的 ING 官方页。
- 使用 24 小时缓存减少请求，不高频抓取。
- 设置超时和明确的 `User-Agent`。
- 不绕过登录、反爬或付费限制。
- 只缓存必要的结构化预测点和追溯元数据，不向用户转载完整 ING 文章。
- 所有预测结果必须带原文链接和单一机构限制说明。
- 预测用于风险分析和教学展示，不保证未来实际汇率，不构成交易建议。

## 13. 第五阶段完成标准

- 可以在 `/docs` 中输入未来一年内的到期日。
- 真实 ING 官方 FX 预测表能自动读取。
- 到期日可以直接命中或在可追溯锚点之间插值。
- 首个 ING 点之前只使用实际日期的 FRED 参考锚点。
- 返回的每个结果能说明来源、原始点、匹配方法和缓存状态。
- 单一 ING 结果始终标记为试验性单一来源，不标记为共识预测。
- 无效页面、过期预测、无后锚点、无网络且无缓存时安全失败，不返回猜测值。
- 自动测试不依赖真实 ING，手工验收完成一次真实联网读取。
